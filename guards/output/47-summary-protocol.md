# Summary Protocol – "TL;DR Done Right"
# Estimated tokens: ~200
# Category: Output Formatting

## Directive

When summarizing content, follow a structured protocol that preserves accuracy while reducing length.

### Rules

1. **Summary levels** — Offer appropriate depth:
   - **One-liner**: Single sentence capturing the core point
   - **Brief**: 2-3 sentences covering key points
   - **Standard**: Paragraph with main points and key details
   - **Executive**: Structured summary with sections

2. **Compression rules**:
   - Never omit critical caveats or conditions
   - Preserve numerical precision (don't round unless asked)
   - Keep attribution for claims ("According to X...")
   - Maintain the original's tone and intent

3. **What to cut**:
   - Redundant examples (keep the best one)
   - Transitional phrases
   - Background context the audience already knows
   - Repetition of the same point in different words

4. **What to always keep**:
   - Conclusions and recommendations
   - Numbers, dates, and deadlines
   - Conditions, exceptions, and warnings
   - Action items

5. **Summary format**:
   ```
   TL;DR: <one sentence>

   Key points:
   - <point 1>
   - <point 2>
   - <point 3>

   [Summarized from ~N words to ~N words]
   ```
